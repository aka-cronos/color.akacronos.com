# Color Palette Generator — Implementation Plan

## Context
Building a color palette generator for color.akacronos.com. The user inputs a base color (HEX or OKLCH), and the tool generates an 11-step Tailwind-style palette (50–950) using the OKLCH color space. APCA contrast scores are displayed per swatch (display-only, no enforcement). A dark mode palette is auto-generated by mirroring the lightness axis. The tool exports CSS custom properties, Tailwind v4 @theme, and JSON tokens.

Stack: Next.js 16, React 19, TypeScript 5, Tailwind v4, App Router. Fresh project — no color libraries installed yet.

---

## Core Algorithms

### 1. Target Lightness Scale (L values, OKLCH 0–1)

| Step | L     | Step | L     |
|------|-------|------|-------|
| 50   | 0.975 | 600  | 0.490 |
| 100  | 0.950 | 700  | 0.390 |
| 200  | 0.900 | 800  | 0.280 |
| 300  | 0.825 | 900  | 0.185 |
| 400  | 0.720 | 950  | 0.130 |
| 500  | 0.600 |      |       |

### 2. Chroma Tapering (smooth-step S-curve)

Peak chroma at steps 300–700 (= base C). Taper with smoothstep toward extremes:
- Below L=0.20 → taper to 25% of base C
- Above L=0.85 → taper to 15% of base C
- Between → full base C

```ts
function smoothstep(t: number) { return t * t * (3 - 2 * t); }
```

### 3. Base Color Placement

After generating 11 steps, check if base L is within ±0.03 of any `TARGET_L`.
- If YES: no extra swatch (base is already represented)
- If NO: insert an additional step named `"base"` with `isBase: true`, sorted by L into the array

### 4. APCA Scoring

```ts
// Use culori converter("lrgb") for linear-light RGB
// Y = 0.2126R + 0.7152G + 0.0722B  (linear values)
// Test: calcAPCA(1.0, bgY) → white text Lc
//       calcAPCA(0.0, bgY) → black text Lc
// Report higher abs(Lc) as the winner
```

Thresholds to display:
- Lc ≥ 75 → Green (Silver body text)
- Lc ≥ 60 → Yellow (general content 16px+)
- Lc ≥ 45 → Orange (large text 32px+)
- Lc < 45 → Red

### 5. Dark Mode Derivation

For each canonical step: `darkL = 1 - TARGET_L[step]`
- Step 50: L 0.975 → dark L 0.025 (near-black background)
- Step 950: L 0.130 → dark L 0.870 (near-white foreground)
- Apply same `computeChroma(darkL, baseC)` tapering
- Keep same H as base color
- Gamut-map and compute APCA per dark swatch

### 6. Gamut Mapping

Use `culori`'s `toGamut("rgb", "oklch")` — binary-search chroma reduction preserving L and H. After mapping, store both the adjusted OKLCH values and the hex string. All APCA computation runs on the gamut-mapped color.

## Key Types

```ts
interface OklchColor { l: number; c: number; h: number }

interface ApcaResult {
  lc: number;
  textColor: "white" | "black";
  passes75: boolean;  // Silver body text
  passes60: boolean;  // General content
  passes45: boolean;  // Large text
}

interface PaletteStep {
  name: string;      // "50" | "100" | ... | "950" | "base"
  oklch: OklchColor;
  hex: string;
  cssOklch: string;  // "oklch(0.600 0.2200 250.0)"
  apca: ApcaResult;
  isBase: boolean;
}

interface Palette {
  name: string;
  steps: PaletteStep[];       // light palette
  darkSteps: PaletteStep[];   // dark palette
  baseColor: OklchColor;
}
```

## Export Formats

**CSS custom properties:**
```css
:root {
  --{name}-50: oklch(0.975 0.0300 250.0);
  ...
}
@media (prefers-color-scheme: dark) {
  :root { --{name}-50: oklch(0.025 0.0075 250.0); ... }
}
```

**Tailwind v4 @theme:**
```css
@theme {
  --color-{name}-50: oklch(0.975 0.0300 250.0);
  ...
}
```

**JSON (Style Dictionary compatible):**
```json
{ "{name}": { "50": { "value": "oklch(...)", "$type": "color" } } }
```

## UI Notes

- `app/page.tsx` is `"use client"`. Use `useMemo` returning `{ palette, error }` together (avoids setState-in-memo anti-pattern).
- Swatch backgrounds use inline `style={{ background: step.cssOklch }}` — do NOT use Tailwind arbitrary values with runtime OKLCH strings.
- `ColorInput` debounces 300ms before triggering palette generation.
- `SwatchCard` renders the "Aa" sample text in the winning text color (white or black) from the APCA result.
